Mobile Dropdown Visual Styling Guide

Component: Mobile Analysis Mode Dropdown | Design System: Material Design 3 + Japanese Color Palette | Date: 2025-01-26

Design Philosophy

The mobile dropdown visual design combines Material Design 3 principles with the Japanese Analyzer's custom color palette to create a cohesive, accessible, and touch-optimized interface that maintains consistency with the desktop experience while optimizing for mobile interaction patterns.

  • Keyboard focus: Clear visual indication for all interactive elements
  • High contrast mode: Border-based indicators for visibility

Implementation Notes

Material Design Token Integration

The dropdown implementation should use the existing Material Design 3 CSS custom properties from the material-theme.css file. Key tokens to reference:

/* Surface colors */ background-color: var(--md-sys-color-surface-container); color: var(--md-sys-color-on-surface); /* Shape tokens */ border-radius: var(--md-sys-shape-corner-medium); /* 12px */ /* Elevation tokens */ box-shadow: var(--md-elevation-1); /* Default state */ box-shadow: var(--md-elevation-3); /* Open state */ /* Typography tokens */ font-family: var(--md-sys-typescale-label-large-font); font-size: var(--md-sys-typescale-label-large-size); /* Japanese color integration */ border-color: var(--grammar-verb); /* #C41E3A */

Performance Optimizations

  • GPU acceleration: Use transform and opacity for animations
  • Will-change optimization: Apply will-change: transform during animations only
  • Touch optimization: Implement proper touch event handling with touch-action: manipulation
  • Bundle size: Use CSS transitions instead of JavaScript animation libraries when possible

Browser Compatibility

  • iOS Safari: Tested on iOS 15+ with proper viewport meta tag
  • Chrome Mobile: Full support for modern Android devices
  • Fallbacks: Progressive enhancement with basic functionality for older browsers
  • CSS Grid: Use with @supports for older browser fallbacks
