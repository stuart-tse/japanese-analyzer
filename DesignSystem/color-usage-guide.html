Rationale:

  • Nouns (Dark Charcoal): Most important content words - use strongest, most readable color
  • Verbs (Primary Red): Action words - use accent color to emphasize importance
  • Adjectives & Particles (Warm Brown): Supporting elements - use warm, secondary color
  • Adverbs & Other (Blue-Gray): Less critical elements - use subtle supporting color

Do's and Don'ts
✓ DO

Use Primary Red (#DF3307) for important actions and verbs to create clear visual hierarchy

✗ DON'T

Use Primary Red for large text blocks or background colors - it can be overwhelming

✓ DO

Use consistent color assignments for parts of speech to help users learn patterns

✗ DON'T

Mix color assignments for the same parts of speech across different screens

✓ DO

Use Light Beige (#DAC8C0) as the primary background to create a warm, comfortable reading environment

✗ DON'T

Use pure black or white backgrounds which can cause eye strain during extended study sessions

✓ DO

Test all color combinations with actual users, especially those with visual impairments

✗ DON'T

Assume color alone conveys information - always provide alternative indicators

React/Tailwind CSS Implementation

Tailwind Configuration

Add these custom colors to your tailwind.config.js file:

// tailwind.config.js module.exports = { theme: { extend: { colors: { // Primary palette 'primary-red': '#DF3307', 'dark-charcoal': '#42433B', 'warm-brown': '#8F7E74', 'light-beige': '#DAC8C0', 'blue-gray': '#9FAEB3', // Parts of speech pos: { noun: '#42433B', verb: '#DF3307', adjective: '#8F7E74', particle: '#8F7E74', adverb: '#9FAEB3', other: '#9FAEB3', }, // Semantic colors semantic: { success: '#8F7E74', warning: '#DF3307', error: '#DF3307', info: '#9FAEB3', } } } } }

CSS Custom Properties

Alternatively, use CSS custom properties for better flexibility:

/* globals.css */ :root { /* Primary palette */ --color-primary: #DF3307; --color-text-primary: #42433B; --color-text-secondary: #8F7E74; --color-background: #DAC8C0; --color-surface: #FFFFFF; --color-border: #9FAEB3; /* Parts of speech */ --color-pos-noun: #42433B; --color-pos-verb: #DF3307; --color-pos-adjective: #8F7E74; --color-pos-particle: #8F7E74; --color-pos-adverb: #9FAEB3; --color-pos-other: #9FAEB3; }

React Component Example

// Token.tsx interface TokenProps { word: string; pos: 'noun' | 'verb' | 'adjective' | 'particle' | 'adverb' | 'other'; furigana?: string; romaji?: string; } export const Token: React.FC<TokenProps> = ({ word, pos, furigana, romaji }) => { return ( <div className={` inline-block m-1 p-3 rounded-lg cursor-pointer transition-all duration-300 hover:-translate-y-1 hover:shadow-lg min-w-[60px] text-center leading-[2.5] border-2 bg-pos-${pos} border-pos-${pos} text-pos-${pos} `}> {furigana && ( <div className="text-xs opacity-80 leading-none mb-1"> {furigana} </div> )} <div className="font-semibold leading-none mb-1"> {word} </div> {romaji && ( <div className="text-xs opacity-70 italic leading-none"> {romaji} </div> )} </div> ); };
Migration from Current Colors

Color Mapping Table

Current Color New Color Usage
#1976d2 (Blue) #DF3307 (Primary Red) Primary actions, headers
#4caf50 (Green) #8F7E74 (Warm Brown) Success states, secondary actions
#333333 (Dark Gray) #42433B (Dark Charcoal) Primary text, nouns
#f5f5f5 (Light Gray) #DAC8C0 (Light Beige) Backgrounds, disabled states
#666666 (Medium Gray) #8F7E74 (Warm Brown) Secondary text, labels

Step-by-Step Migration

  1. Update your Tailwind configuration or CSS variables
  2. Replace all instances of old primary color (#1976d2) with new primary red (#DF3307)
  3. Update text colors to use dark charcoal (#42433B) for primary text
  4. Change background colors from gray to light beige (#DAC8C0)
  5. Update parts of speech token colors according to the new assignments
  6. Test accessibility and contrast ratios
  7. Update any hardcoded color values in components
  8. Verify the design across different devices and browsers
